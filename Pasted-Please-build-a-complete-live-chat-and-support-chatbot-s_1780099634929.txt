Please build a complete live chat and support chatbot system into the project. Do all steps in order and do not break any existing functionality.

STEP 1 - Add chat database schema:
Open lib/db/src/schema and create or update the messages/chat schema to add a support_chat table with these fields:
  id: uuid primary key defaultRandom
  sessionId: text not null (for non-logged-in visitors using a random session ID)
  userId: uuid nullable (set if the visitor is a logged-in user)
  visitorName: text nullable
  visitorEmail: text nullable
  message: text not null
  fromAdmin: boolean default false
  isRead: boolean default false
  createdAt: timestamp defaultNow

Also add a support_chat_sessions table:
  id: uuid primary key defaultRandom
  sessionId: text not null unique
  userId: uuid nullable
  visitorName: text nullable
  visitorEmail: text nullable
  status: text default "open" (values: open, closed, resolved)
  lastMessageAt: timestamp defaultNow
  createdAt: timestamp defaultNow

Run: cd lib/db && npx drizzle-kit push

STEP 2 - Add support chat API routes:
Create a new route file artifacts/api-server/src/routes/support-chat.ts with these routes:

POST /support-chat/session
- Creates or retrieves a chat session using sessionId from request body
- Accepts: sessionId, visitorName (optional), visitorEmail (optional), userId (optional)
- Returns the session object

POST /support-chat/message
- Accepts: sessionId, message, visitorName (optional), visitorEmail (optional)
- Saves the message to support_chat table with fromAdmin false
- Returns the saved message

GET /support-chat/messages/:sessionId
- Returns all messages for a session ordered by createdAt asc
- Polls every 3 seconds from frontend

POST /support-chat/admin/reply
- Protected with admin auth middleware
- Accepts: sessionId, message
- Saves message with fromAdmin true
- Returns the saved message

GET /support-chat/admin/sessions
- Protected with admin auth middleware
- Returns all chat sessions ordered by lastMessageAt desc
- Include unread message count per session
- Include last message preview

PATCH /support-chat/admin/sessions/:sessionId/status
- Protected with admin auth middleware
- Accepts: status (open, closed, resolved)
- Updates session status
- Returns updated session

GET /support-chat/admin/sessions/:sessionId/messages
- Protected with admin auth middleware
- Returns all messages for a session

Register the new route file in artifacts/api-server/src/app.ts or index.ts
