STEP 1 - Add merchant application data model:
Open lib/db/src/schema/p2p.ts and add a new table p2p_merchant_applications with these fields:
  id: uuid primary key defaultRandom
  userId: uuid not null (FK to users)
  legalName: text not null
  email: text not null
  country: text not null
  paymentMethod: text not null (values: etransfer_email or bank_account)
  etransferEmail: text nullable
  bankReceivingDetails: jsonb nullable
  description: text not null
  status: text not null default "pending" (values: pending, approved, rejected)
  rejectionReason: text nullable
  approvedAt: timestamp nullable
  createdAt: timestamp defaultNow

Also add p2p_notifications table if not already present with fields:
  id: uuid primary key defaultRandom
  userId: uuid not null
  type: text not null (values: p2p_deposit)
  asset: text not null
  amount: text not null
  reference: text not null
  instructions: text nullable
  isRead: boolean default false
  createdAt: timestamp defaultNow

Then run: cd lib/db && npx drizzle-kit push

STEP 2 - Add merchant application API routes:
In artifacts/api-server/src/routes/p2p.ts add these routes:

POST /p2p/merchant/apply
- Protected with requireAuth
- Accepts: legalName, email, country, paymentMethod, etransferEmail, bankReceivingDetails, description
- Validates with Zod: all fields required except etransferEmail and bankReceivingDetails which depend on paymentMethod
- Checks if user already has a pending or approved application and returns 400 if so
- Inserts new application record with status pending
- Returns the created application

GET /p2p/merchant/status
- Protected with requireAuth
- Returns the user's most recent merchant application including status, rejectionReason, and approvedAt
- Returns null if no application exists

Also update the GET /auth/session response to include merchantStatus field (pending, approved, rejected, or null) based on the user's application status, and isMerchant boolean from the user record.

STEP 3 - Add admin merchant management routes:
In the admin routes file add these routes all protected with admin auth middleware:

GET /admin/p2p/merchants
- Accepts optional query param status (pending, approved, rejected)
- Returns list of all merchant applications with user details joined

PATCH /admin/p2p/merchants/:userId/decision
- Accepts: action (approve or reject), note (optional), rejectionReason (required if action is reject)
- If action is approve: set application status to approved, set approvedAt to now, set isMerchant true on the user record, store optional welcome note
- If action is reject: set application status to rejected, store rejectionReason on the application
- Returns updated application

DELETE /admin/p2p/merchants/:userId
- Revokes merchant status: set isMerchant false on user record, set application status back to rejected with rejectionReason "Merchant status revoked by admin"
- Returns success message

Add Zod schemas for all request bodies and regenerate client hooks after.
